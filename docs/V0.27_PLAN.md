## Feature 5 — Fix the copy/paste mojibake (do FIRST)

**Root cause (verified):** `ptyService.ts` spawns the PTY with `TERM`/`COLORTERM`
but **no locale**. A macOS app launched from Finder inherits no `LANG`/`LC_*`, so
the shell + Claude fall back to C/POSIX and misread multibyte UTF-8 **input**
(pasted emoji / em-dashes → `üéâ`, `‚Äî`), while output still renders.

- **Modify** `src/main/services/ptyService.ts`: extract a pure helper
  `ensureUtf8Locale(env, platform)` and call it on the spawn `env`.
  - darwin: if none of `LC_ALL`/`LC_CTYPE`/`LANG` matches `/utf-?8/i`, set
    `env.LC_CTYPE = 'UTF-8'` (valid BSD/macOS value; sets encoding only, not
    language). Leave Linux/Windows untouched (Linux inherits `LANG` from the
    launching shell; Windows ConPTY is UTF-16).
- **Create** `tests/unit/ptyLocale.test.ts`: no-locale darwin → `LC_CTYPE=UTF-8`;
  `LANG=en_US.UTF-8` present → unchanged; non-darwin → unchanged; existing
  `LC_CTYPE=UTF-8` → unchanged.
- IPC: none. Security: none (env only).

## Feature 6 — Multi-select in the file tree + drag REAL files out

**Two parts.** (a) Cmd/Ctrl-click multi-select with an action bar; (b) dragging a
selected file (or files) into another app drops the **actual file**, not its path.

- **Modify** `src/renderer/src/components/files/FileTree.tsx`:
  - Selection state: `Set<string>` of relPaths. Plain click = single; Cmd/Ctrl+click
    = toggle; Shift+click = range over the flattened visible list.
  - Action bar when ≥1 selected: **Send to Claude** (paste the quoted absolute
    paths — or `@relpath` refs — into the focused pane via `paneWriters`), **Copy
    paths**, **Reveal**. (Reuses existing `paneWriters` + `fs:reveal`.)
  - `onDragStart` on a row: if the row is part of the selection, drag the whole
    selection. Call the new `fs:startDrag` so the OS does a native file drag.
- **IPC (new):** `'fs:startDrag': (req: { relPaths: string[] }) => Result<void>`
  - add to `InvokeChannels` + `INVOKE_CHANNELS` in `src/shared/ipc.ts`.
  - **Handler** in `src/main/ipc/handlers/fs.ts`: zod `{ relPaths: string[] (max ~50) }`;
    resolve each through **`pathJail`** to an absolute path inside the project
    (reject anything outside — same jail used by all fs handlers); then
    `event.sender.startDrag({ files: absPaths, icon })`. `icon` = a small bundled
    `nativeImage` (Electron requires a non-empty drag icon).
  - **Create** `build/dragicon.png` (a tiny generic file glyph) bundled via
    electron-builder `files`/`extraResources`, loaded with `nativeImage.createFromPath`.
- **Create** `src/renderer/src/components/files/fileSelect.ts` (pure):
  `toggle/range/single` selection reducers over a flat path list.
- **Create** `tests/unit/fileSelect.test.ts`: single/toggle/range/clear logic.
- Security: `fs:startDrag` is **jailed** — only files inside the open project can be
  dragged out; the user initiates the drag. No symlink escape (pathJail is
  symlink-safe).

## Feature 2 — Restore terminal scrollback across a full quit

**Honest scope:** processes die on quit (no daemon — out of scope, IDE-creep). We
**serialize each terminal's on-screen scrollback** and restore it as read-only
history on relaunch, then offer one-click `claude --resume` (Feature 3). Nothing
visual is lost; the live session is not resurrected.

- **Modify** `terminalPool.ts`: load `@xterm/addon-serialize` (already a dep). Add
  pool method `serialize(maxLines)` → `serializeAddon.serialize({ scrollback })`.
  On create, if restored content exists for the leaf id, `term.write(restored)`
  **before** `startPty()` so it shows as dim history, then write a thin separator
  line (`\x1b[2m──── restored ────\x1b[0m`).
- **Renderer save trigger:** in the workspace layer, serialize all persistent
  pooled terminals (cap ~1500 lines each) on `window` `beforeunload` **and** on a
  30s debounce, sending them to main.
- **IPC (new):**
  - `'terminal:saveBuffers': (req: { buffers: { leafId: string; data: string }[] }) => Result<void>`
  - `'terminal:loadBuffers': (req: void) => Result<{ leafId: string; data: string }[]>`
  - both added to `InvokeChannels` + `INVOKE_CHANNELS`.
- **Create** `src/main/services/terminalBufferStore.ts`: persists to
  `userData/terminal-buffers-<windowKey>.json`; prunes leaf ids no longer present;
  caps total size (e.g., 4 MB) by evicting oldest. Pure trim/cap helpers exported.
- **Modify** `src/main/ipc/handlers/` (new `terminal.ts` or fold into `pty.ts`) +
  `register.ts`.
- Setting `terminal.restoreScrollback` (default `true`).
- **Create** `tests/unit/terminalBufferStore.test.ts`: line-cap trim, size-cap
  eviction, prune-missing-leaves.
- Security: writes only to `userData` (app's own dir), reads only its own file. The
  buffer is terminal output the user already saw; stored locally, never sent.

## Feature 7 — Checkpoint navigator + safe `/rewind` hand-off (the big one)

**What's reliable vs not** (from Claude Code docs research):
- ✅ Checkpoints = one per user prompt; the prompt list is reconstructable from the
  session transcript (`~/.claude/projects/<slug>/<session>.jsonl`).
- ❌ **No programmatic rewind** — no CLI/MCP/hook/SDK. Restore is ONLY the
  interactive `/rewind` TUI. Snapshot files exist at `~/.claude/file-history/{id}/`
  but the versioning is undocumented and restoring them ourselves would desync code
  from Claude's conversation state. So we **drive Claude's own `/rewind`** and let
  the user confirm the mode.

**Design:** a tab-level **History rail** beside the terminal (toggled from
`.pane__controls`; side = left/right via setting), listing your prompts newest-first.

- **Create** `src/main/services/sessionHistoryService.ts`: tails the transcript for
  the active project (reuse the usage/agent tail mechanics) and extracts **user
  prompts** → `{ index, ts, text, preview }[]` for the most-recently-active session
  matching a given cwd. Pure parser `parseUserPrompt(line)` + `buildHistory(events)`.
- **IPC (new):**
  - `'session:getHistory': (req: { cwd: string }) => Result<SessionHistory>`
  - `'session:changed': SessionHistory` (event → live updates while you work)
  - added to the maps + both runtime allowlists; new `SessionHistory` type in
    `src/shared/types.ts`.
  - **Handler** in a new `src/main/ipc/handlers/session.ts` + `register.ts`; service
    started in `index.ts` next to the others.
- **Renderer:**
  - **Create** `src/renderer/src/state/useSessionHistoryStore.ts` (mirror
    `useAgentStore`: `session:changed` + `session:getHistory`).
  - **Create** `src/renderer/src/components/history/HistoryRail.tsx`: the list; each
    row = prompt preview + relative time + a 3-dots/right-click menu.
  - **Click a row → scroll to it:** call the focused pane's `findNext(promptFirstLine)`
    (SearchAddon, already loaded). If not found in scrollback → toast "scrolled out
    of view". (Pool exposes `scrollToMatch(text)`.)
  - **3-dots / right-click → "Rewind to here":** the **safe hand-off** —
    1. `paneWriters.write(leafId, '/rewind\r')`.
    2. Detect Claude's rewind menu in the buffer (new parser, like `parseAsk`),
       confirm the entry count matches, then send `↓`×N (paced, via the existing
       munu key-queue pattern) to land on the clicked checkpoint, and **stop** —
       the user picks code/conv/both in Claude's own menu.
    3. If the menu can't be confidently detected/mapped → **fallback**: just leave
       `/rewind` open for the user to navigate. Never auto-confirm a restore.
  - Menu also offers "Copy prompt" and "Scroll here".
- **Modify** `PaneTree.tsx` / the tab layout to render the rail beside `PaneTree`
  (resizable, collapsible); toggle button in `.pane__controls` (lucide `Milestone`
  or `ListTree`). Settings: `sessionHistory.{ enabled, side }`.
- **Create** `src/renderer/src/components/terminal/rewindParse.ts` (pure): detect +
  count entries in Claude's `/rewind` menu from buffer text; map clicked index →
  arrow count. Best-effort, with an explicit "confident?" boolean.
- **Create** `tests/unit/sessionHistory.test.ts` (parse user prompts, build list,
  cwd→session selection) and `tests/unit/rewindParse.test.ts` (menu detection +
  arrow math + the not-confident fallback).
- Security: transcript reads are read-only; `/rewind` + arrows are user-initiated;
  the destructive choice happens in Claude's own UI. Mark output as untrusted text
  (React auto-escapes the prompt previews).

## Acceptance criteria

- **#5:** pasting `🎉 — ✅` into Claude shows them correctly (no `üéâ`).
- **#3:** the two buttons run `claude` / `claude --resume` in the pane.
- **#1/#4:** selecting shows the toolbar; "Send to Claude" drops the text into the
  pane's prompt (multi-line intact); selecting alone does not auto-copy; ⌘C still copies.
- **#6:** Cmd-click selects multiple; dragging them into Finder/Chrome drops the real
  files; "Send to Claude" inserts their paths; drag is jailed to the project.
- **#2:** quit + reopen → each terminal shows its prior scrollback (dim history) +
  a fresh prompt; `claude --resume` continues the conversation.
- **#7:** the rail lists your prompts; clicking scrolls to one; "Rewind to here"
  opens Claude's `/rewind` at that checkpoint and lets you choose the mode; if the
  menu can't be mapped, it just opens `/rewind` (never mis-restores).
- All gates green; new unit tests pass; no regressions in terminal/munu/usage/activity.

## Risks / honest caveats

1. **#7 rewind navigation is best-effort** (TUI parsing, like munu) — fallback never
   auto-confirms a destructive restore.
2. **#2 restores visuals, not live processes** — by design; `claude --resume` bridges it.
3. **#5** assumes the locale is the cause (very likely on macOS); a 2-min repro
   confirms before/after. If a residual case remains, normalize the paste encoding too.
4. **#7 cwd→session mapping** picks the most-recently-active transcript for the pane's
   project (ambiguous with multiple concurrent sessions in one folder) — documented.
5. **Scope:** this is large; if any step risks the release, it ships behind its
   setting (off) rather than blocking the rest.
