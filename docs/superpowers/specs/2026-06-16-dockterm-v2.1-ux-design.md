# DockTerm v2.1 — UX Power Pack (Design Spec)

**Status:** approved direction, 2026-06-16. Builds on V2 (v0.4.1).

## Goal
Five UX upgrades: terminal selection auto-scroll + Mac scroll shortcuts, drag-a-file-onto-terminal to insert its path, first-class file viewing (images + binaries), and a multi-window / per-pane-project model (new window = Cursor-style welcome; panes can target different projects; the dock follows the focused pane). Identity unchanged: local-only, no telemetry/cloud.

## 1. Terminal scrolling (useTerminal)
- **Selection auto-scroll:** xterm.js auto-scrolls during a drag-selection past the viewport; ensure nothing in the pane layout blocks it (pointer events / overflow). Verify and unblock.
- **Auto-scroll to bottom on input/output:** keep xterm `scrollOnUserInput` on so typing returns you to the prompt (no shortcut needed).
- **Mac shortcuts** via `term.attachCustomKeyEventHandler`: ⌘↓ → `scrollToBottom()`, ⌘↑ → `scrollToTop()`, Shift+PageUp/PageDown → page; these are intercepted (not sent to the PTY). Replaces the obscure Ctrl+End.

## 2. Drag a file → terminal inserts its path
- **File-tree rows draggable:** `onDragStart` sets `dataTransfer` with `application/x-dockterm` = JSON `{ path: <abs>, type: 'file'|'dir' }` and `text/plain` = abs path. Absolute path needs the active project root in the renderer (see §5: `project:setActiveRoot` returns the resolved root; stored as `useAppStore.activeRoot`). Abs path = `activeRoot + '/' + relPath`.
- **Terminal pane = drop target** (in `TerminalPane`): `onDragOver` preventDefault; `onDrop`:
  - From the tree, a **file** → paste the abs path (quoted if it contains spaces) into the PTY.
  - From the tree, a **folder** → retarget this pane to that folder (§5).
  - From **Finder/Explorer** (`e.dataTransfer.files`, Electron `File.path`) → paste quoted, space-separated abs paths.
- Paste = `pty:write` the text (no trailing newline).

## 3. File viewing — images + binaries
- Editor tab gains `kind: 'text' | 'image' | 'binary'`. Classification by extension on open: images = png/jpg/jpeg/gif/webp/svg/bmp/ico/avif → `image`; otherwise if `fs:readFile` returns `binary`/`too-large` → `binary`; else `text`.
- New IPC `fs:readDataUrl(relPath) → { dataUrl, size }` — jailed, size-capped (25 MB), builds `data:<mime>;base64,<...>` from the file (mime by extension).
- New IPC `fs:openPath(relPath) → void` — `shell.openPath(absInsideRoot)` to open a file in its OS default app.
- `EditorPane` renders by kind: `text` → Monaco (today), `image` → **ImageViewer** (`<img>` fit-to-view + click-to-zoom, checkerboard backdrop), `binary` → **BinaryCard** (name, size, **Reveal in Finder**, **Open externally**). Editor tabs show all kinds.

## 4. Multi-window + per-pane projects
- **New window = Cursor-style welcome:** only the **primary** window restores `lastProjectPath`; secondary (⌘N) windows start **project-less** and show the `EmptyState`, which offers **Open Folder** + **recent projects**. (`useAppStore.init`: restore last project only when `isPrimary`.) Ensure `EmptyState` has an Open-Folder button + recents list.
- **Per-pane working directory** (already built): each pane/leaf has its own `cwd`; the dock (Files/Git/Review/Info/MCP/Skills) targets the **focused pane's project** via `project:setActiveRoot` → `resolveProjectRoot`.
- **Make `project:setActiveRoot` return the resolved root** (`Result<{ root }>`); the Shell focus effect stores it as `useAppStore.activeRoot` so the file tree can build absolute paths (§2) and show the right root.
- **Point a pane at another project — two ways:**
  - **"New tab in folder…":** a folder picker (`project:openDialog`) → `useWorkspaceStore.open(pickedFolder)` opens a new tab whose terminal runs there. Surfaced as a split-button/menu on the tab strip's `+`.
  - **Drag a folder from the tree onto a pane → retarget that pane:** `useWorkspaceStore.retargetLeaf(leafId, cwd)` updates the leaf's `cwd` + title; the `TerminalView` is keyed by `leaf.id + ':' + leaf.cwd` so it remounts and respawns the shell in the new folder. Layout helper `setLeafCwd(root, leafId, cwd)` (pure, unit-tested).
- Result: 2×3 grid of terminals, each possibly a different project; focusing one switches the dock to its project — the model the user asked for.

## 5. IPC / data-model changes
- `project:setActiveRoot`: `(PathReq) => Result<{ root: string }>` (returns resolved root).
- `fs:readDataUrl`: `(RelPathReq) => Result<{ dataUrl: string; size: number }>`.
- `fs:openPath`: `(RelPathReq) => Result<void>`.
- Editor store tab: add `kind`. Layout: add `setLeafCwd`. `useAppStore`: add `activeRoot: string | null`.
- All new channels added to the runtime allow-list; renderer stays sandboxed; path-jail enforced on every new fs IPC.

## 6. Security
- New fs IPCs are path-jailed within the active root (realpath + prefix). Image data URLs are local-file reads only, size-capped. No remote content; CSP unchanged. `shell.openPath` only opens files inside the jailed root.

## 7. Testing
- Pure/unit: `setLeafCwd` (layout), absolute-path building, extension→kind classification, mime-by-extension. Keep typecheck + vitest green.
- Manual: selection auto-scroll, drag-file/folder, image/binary tabs, ⌘N welcome, per-pane dock switching, 2×3 mixed-project grid.

## 8. Build order
A. Scroll behavior (1) — small, isolated.
B. File viewing (3) — IPCs + ImageViewer/BinaryCard + editor `kind`.
C. `activeRoot` plumbing + drag-file-to-terminal path insert (2).
D. Multi-window welcome + per-pane projects: new-window restore gate, EmptyState welcome, new-tab-in-folder, drag-folder-retarget (4).

## 9. Non-goals
No remote/SSH, no syntax for non-text beyond image preview, no Quake drop-down (separate), no tabs-relocation (separate).
